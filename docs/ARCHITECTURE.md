## 技术栈选型

### 前端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| **Next.js** | 14.x | 全栈框架，App Router |
| **React** | 18.x | UI 组件库 |
| **TypeScript** | 5.x | 类型安全 |
| **Tailwind CSS** | 3.x | 样式框架 |
| **Radix UI** | 最新 | 无障碍组件基础 |
| **Zustand** | 4.x | 客户端状态管理 |
| **React Query** | 5.x | 服务端状态管理 |

### 后端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| **Next.js API Routes** | 14.x | API 服务 |
| **Prisma** | 5.x | ORM |
| **PostgreSQL** | 15.x | 主数据库 |
| **Redis** | 7.x | 缓存/限流/会话 |
| **Zod** | 3.x | 运行时类型校验 |

### 基础设施

| 技术 | 用途 |
|------|------|
| **Vercel** | 部署平台 |
| **Cloudflare** | CDN / DDoS防护 |
| **Upstash** | Redis 云服务 |
| **Supabase/Neon** | PostgreSQL 云服务 |

## 模块通信机制

### 模块间通信原则

```
┌─────────────┐                      ┌─────────────┐
│  模块 A     │                      │  模块 B     │
├─────────────┤                      ├─────────────┤
│             │    ❌ 禁止直接调用    │             │
│             │  ──────────────────► │             │
│             │                      │             │
│             │    ✅ 通过事件总线    │             │
│             │  ──────────────────► │             │
└─────────────┘                      └─────────────┘
        │                                    ▲
        │                                    │
        ▼                                    │
┌─────────────────────────────────────────────────────┐
│                    事件总线 (Event Bus)              │
│  - article:created    - article:published           │
│  - article:verified   - article:deprecated          │
│  - agent:api-called   - agent:quota-warning         │
└─────────────────────────────────────────────────────┘
```

### 状态共享原则

```
┌─────────────────────────────────────────────────────┐
│                  全局状态 (Global State)             │
│  - 当前用户/Agent    - 语言设置                      │
│  - 系统配置          - 功能开关                      │
└─────────────────────────────────────────────────────┘
        │                    │
        ▼                    ▼
┌─────────────┐      ┌─────────────┐
│  模块 A     │      │  模块 B     │
│  订阅状态   │      │  订阅状态   │
└─────────────┘      └─────────────┘
```

## 数据流架构

### 请求处理流程

```
客户端请求
    │
    ▼
┌─────────────────┐
│   Middleware    │ ◄── 认证/限流/日志
└─────────────────┘
    │
    ▼
┌─────────────────┐
│   API Route     │ ◄── 参数校验/路由分发
└─────────────────┘
    │
    ▼
┌─────────────────┐
│    Service      │ ◄── 业务逻辑
└─────────────────┘
    │
    ├──────────────┐
    ▼              ▼
┌─────────┐  ┌──────────┐
│   DB    │  │  Cache   │
└─────────┘  └──────────┘
    │
    ▼
┌─────────────────┐
│  事件发布       │ ◄── 领域事件
└─────────────────┘
    │
    ▼
响应返回
```

### AISO 双模渲染流程

```
请求进入详情页
    │
    ▼
┌─────────────────┐
│ 判断请求来源     │
│ User-Agent /    │
│ Accept header   │
└─────────────────┘
    │
    ├─── 人类用户 ────┐
    │                 ▼
    │           ┌─────────────┐
    │           │ SSR 渲染    │
    │           │ HTML + CSS  │
    │           └─────────────┘
    │
    └─── AI Agent ────┐
                      ▼
                ┌─────────────┐
                │ 原始输出    │
                │ Markdown/   │
                │ JSON        │
                └─────────────┘
```

## 安全架构

### API 安全策略

```
┌─────────────────────────────────────────────────────┐
│                    安全层                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  速率限制 (Rate Limiting)                           │
│  ├── IP 维度: 100 req/min (匿名)                    │
│  ├── API Key 维度: 1000 req/day (认证)              │
│  └── 熔断机制: 异常检测自动触发                      │
│                                                     │
│  认证授权 (Auth)                                    │
│  ├── 公开 API: 无需认证 (限额内)                    │
│  ├── 内部 API: 服务间密钥                           │
│  └── 管理后台: Session + RBAC                       │
│                                                     │
│  数据安全                                           │
│  ├── API Key 哈希存储                               │
│  ├── 敏感字段加密                                   │
│  └── SQL 注入防护 (Prisma 参数化)                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```
