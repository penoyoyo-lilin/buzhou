# 巡检中心每日增量执行结果提示方案

## 问题确认

- 当前“每日增量巡检”按钮已经会在同一次请求中完成入队和执行。
- 但页面没有把这次执行结果直接反馈给用户，用户必须自行看表格刷新或网络响应。
- 现有页面没有统一 toast 机制，本次不引入新的全局依赖，只补最小的页面内反馈。

## 修复范围

限制在 3 个文件内：

1. `src/app/admin/inspections/page.tsx`
2. `tests/integration/api/article-inspection.routes.test.ts`
3. `docs/task-plans/admin-inspection-daily-feedback-20260313.md`

## 修复思路

1. 在巡检中心页面增加一个本地反馈区块：
   - 仅在手动执行 `daily` 或 `process` 成功后显示
   - `daily` 展示“已入队 X 条，已处理 Y 条”
   - `process` 展示“已处理 Y 条”
2. 反馈在下一次手动操作成功后覆盖更新；请求失败时不显示成功提示。
3. 保持现有接口结构不变，不新增新的后端字段。

## 验收标准

- 点击“每日增量巡检”成功后，页面能直接看到“已入队 X 条，已处理 Y 条”。
- 点击“处理 5 条”成功后，页面能直接看到“已处理 Y 条”。
- 刷新列表后提示仍保留在当前页面状态中，直到下一次手动操作覆盖。

## 测试计划

- 复用现有集成测试，继续保证接口返回 `enqueued` / `processed`。
- 页面交互实测：
  - `daily` 成功后提示文案正确
  - `process` 成功后提示文案正确
  - 失败时不显示误导性的成功提示

## 风险与关注点

- 这是页面本地状态提示，不是持久化通知；刷新整页后会消失。
- 本次不引入 toast/toaster，避免扩大范围到全局 UI 基础设施。
