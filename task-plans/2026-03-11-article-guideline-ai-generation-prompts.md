# 任务方案：文章撰写规范补充 AI 生成提示词

## 目标
在“文章撰写规范和指南文档”中新增明确建议：
- `relatedIds`、`qaPairs`、`tags`、`keywords` 建议由平台内部 agent 调用 AI 自动生成。
- 提供可直接复用的提示词模板，便于 internal agent 调用。

## 修改范围
1. 主文档（文章撰写规范与指南）
   - 增加“AI 辅助字段生成”章节
   - 在字段说明中标注建议自动生成字段
2. 补充提示词模板
   - `relatedIds` 生成提示词
   - `qaPairs` 生成提示词
   - `tags` 与 `keywords` 生成提示词

> 约束：仅修改不超过 2 个文件。

## 依赖与前置
- 现有字段契约（`Article`、`QAPair`、`relatedIds`、`tags`、`keywords`）
- internal API 当前字段命名约定（下划线 domain）

## 验收标准
1. 文档中明确声明上述字段“推荐由内部 agent 通过 AI 生成”。
2. 每个字段至少有 1 套可直接粘贴使用的提示词模板（含输入、输出约束）。
3. 模板输出格式与现有字段结构一致（例如 `qaPairs` 为中英双语结构）。
4. 文档整体不引入连字符 domain 等旧规范回退。

## 风险
- 当前工作树未包含 `docs/` 目录，可能需要在可见路径中创建/维护该文档副本。
